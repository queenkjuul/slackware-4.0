Linux-PAM

This is the primary distribution site for the Linux-PAM (Pluggable Authentication Modules for Linux) project. (The ftp version of this site is ftp://ftp.kernel.org/pub/linux/libs/pam/.)

Mirrors of this site include:

  • ftp://ftp.man.szczecin.pl/pub/security/SystemTools/pam
  • ftp://src.doc.ic.ac.uk:/packages/linux/ftp.kernel.org/libs/pam
  • ftp://ftp.ras.ru/pub/mirrors/ftp.kernel.org/pub/linux/libs/pam

Things to be found here are documentation and source code for Linux-PAM.

Documentation

  • Some documentation:
    • The beginnings of a FAQ
    • Online documentation
    • PostScript documentation:
      • For sys-admins, application writers and module developers.
    • a gzip'd tar file of html+PostScript+text documentation:
      • Linux-PAM-doc.tar.gz
    • A current list of modules (and application patches) available or under development.
    • Some more information on Linux-PAM from Red Hat (who've been including PAM in their distribution since they released Red Hat Linux 3.0.4). If you're confused about what PAM is, you might like to look here. Red Hat gets to deal with a lot of users, so they have taken some time to provide a gentle (amusing) overview.
  • FindMail have taken on the task of archiving the Linux-PAM mailing list.
  • Here is a working document attempting to generalize PAM to support client<->server programming models. The original DCE/OSF-RFC as proposed by the people at SunSoft.
  • Further documentation courteously made available by Sun Microsystems,
    • The manual for pam.conf
    • The manual for pam
    • The manual for pam_sm module functions
    • The manual for pam_get_user
    • The XSSO spec which is X/Open's attempt to absorb PAM into something bigger (draft from 1997 courteously made available to us by Vipin).
  • Some more information for you if you want to use both Solaris and Linux's PAM implementation.

Here we list the people that have found bugs in the Linux-PAM distribution. It is thanks to their efforts that Linux-PAM is a safer piece of software.

Source code

PAM for me is a hobby. For this reason I do not supply pre-compiled binaries and am probably not the most appropriate person to seek support from. [Feedback is welcome -- see below for email addresses.] If you are looking for a stable version of this code, you should look to versions that are included in standard distributions. Caldera and Red Hat have PAM support integrated into their systems. They also offer user support.
